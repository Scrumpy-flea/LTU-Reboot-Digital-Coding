Introduction to CSS

What have I learnt?

We compared a website made with just html and then saw it with the CSS added. CSS stands for Cascading Style Sheet. Css is the styling rules that apply to your page. It's stored and written in a seperate document, which we discovered how to connect that to our html. I have learned the different CSS class, id and tag selectors and how to apply properties and give them values. We also looked into colors, fonts and all the avilable options. Also some simple CSS properties and how to use visual studio code to look for different available values. We used Dev tools to inspect the box model on a bbc webpage. Alex drew boxes to help us visualise the elements and spacing they require, While he explained the box model. Margins, padding and borders can affect the way elements layout on a page.

How did I use this in practice?

We have been creating our own google homepage. Today we started creating our own css file for our google. We linked the stylesheet to the html file and added a couple of rules.

What did I find hard?

I found the box model explinations helped me get a clearer understanding.

11/05/2021

More CSS

What have I learnt?

We discussed layout in CSS. We discussed layouts for different size screens. The difference between reactive and responsive. Reactive is no longer used in practise, but it was essentially writing seperate code and files for different size screens. Responsive is using Media Queries to declare different rulesets for different size screens. Meaning webpages are easier to manage.
Css code should be stored in a seperate file to your HTML. However, inline styles can be applied 'Above the fold'. As 50% of users will leave a page if it hasn't loaded in 2.5 seconds. As the styles are in the html document the browser will be able to load them immediatly.
Flexbox is a easy way to plan layouts in your css design.

How did I use this in practice?

We put flexbox into practise on our google homepages. We needed to set the navigation links to sit in a evenly spaced line accross the top of the page.

What did I find hard?

I found the CSS flexbox a little confusing, espically the properties and values and how they affect the main and cross axis. Eg, justify-content and align-items.
After playing flexbox froggy, I feel like I have a better understanding when it comes to flexbox layout. I also looked into flexbox on w3 schools.
